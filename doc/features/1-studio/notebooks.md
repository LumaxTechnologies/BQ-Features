# Notebooks

## What it is

**Notebooks** in BigQuery Studio are Colab Enterprise notebooks (Python, SQL, DataFrames) with runtimes. They are versioned and shareable like saved queries. **Spark notebooks** are available for Apache Spark workloads.

## Demo doc

Use the finance demo notebook generated by the CLI (`bq_studio_demos/notebooks/finance_analytics_demo.ipynb`) to load demo tables, compute returns, and plot PnL. Upload it to Studio and run it against `bq_studio_demo`.

## Demo material

1. **Upload the demo notebook**  
   In BigQuery Studio → **Explorer** → **Notebooks** → **View actions** → **Upload to Notebooks**. Select `bq_studio_demos/notebooks/finance_analytics_demo.ipynb`. Set name and region, then **Upload**. Or use `bqdemo upload-to-studio` to automate.

2. **Open and set project**  
   Open the uploaded notebook. Ensure the first code cell uses your project ID (the notebook uses `bigquery.Client(project='...')` and a dataset variable). Update if your dataset is not `bq_studio_demo`.

3. **Run: load tables**  
   Run the cell that loads `daily_prices`, `portfolio_holdings`, and `pnl_daily` into DataFrames. Check the head() output.

4. **Run: time series and returns**  
   Run the cell that computes `prices['return']` and plots close prices for a few symbols. Confirm the chart uses demo data.

5. **Run: PnL bar chart**  
   Run the cell that aggregates PnL by date and plots a bar chart. This uses `bq_studio_demo.pnl_daily`.

6. **Save and share**  
   Save the notebook (it will version). Use **Share** to grant **Code Viewer** or **Code Editor** to others.

## Demo data used

- Dataset: `bq_studio_demo`.  
- Tables: `daily_prices`, `portfolio_holdings`, `pnl_daily`.  
- Notebook: `bq_studio_demos/notebooks/finance_analytics_demo.ipynb`.
